# P3.1 — Ranged enemy (Acid Spitter) + enemy-projectile system

> First milestone of Phase 3. Goal: introduce a **ranged** enemy behavior and a **pooled enemy projectile**, proving the FSM extends beyond melee and fixing the floating-ledge perch exploit. Built **placeholder-first** — real spitter art is dropped in later via the swap-point.

## Context
Today all four zombies share one melee behavior in `Enemy.preUpdate`: PATROL → CHASE → ATTACK (touch damage) → RETREAT → HURT → DEAD. There is no `aiProfile` concept yet. The RETREAT state is a *stopgap* that just makes melee enemies back off from a perched player — it doesn't threaten them. P3.1 adds a genuinely different behavior via a new `aiProfile: 'ranged'` branch plus an arcing acid projectile, so a player standing on a ledge is now actually in danger.

## Scope guard
**In P3.1:** an `aiProfile` field on enemy data (existing zombies → `'melee'`, behavior unchanged); a `'ranged'` branch in the FSM (kite to a preferred distance, fire an arcing projectile instead of touching); a pooled `AcidProjectile` (green, ballistic arc, damages the player, splats on impact/ground/expiry); one spitter placed in the level to test — specifically positioned so it must hit a **perched** player; placeholder art (green blob spitter, green acid projectile, green splat particle).
**NOT in P3.1:** weapons/switching (P3.2), scrap/shop (P3.3), other new enemy types (P3.4), real spitter art (later AI-gen), terrain line-of-sight occlusion/raycasting (keep LOS simple — range + facing only), player/melee-zombie/terrain/HUD changes.

## Design decisions
- **`aiProfile` is the extension point.** In `spawn()`, read `this.aiProfile` from the enemy's config entry (default `'melee'`). The melee path stays **byte-for-byte unchanged** — only a new branch is added for `'ranged'`. This is the pattern P3.4 reuses for runner/tank/flyer, so keep it clean and generic.
- **Ranged behavior (the spitter):** detection like melee, but:
  - **CHASE → kite:** maintain a `preferredRange` band — move toward the player if farther than `preferredRange.max`, back away if closer than `preferredRange.min`, else hold and face the player. (It doesn't need to be adjacent — it wants distance.)
  - **ATTACK:** when the player is within `firingRange` and roughly in front, and the cooldown is ready, play the attack anim and **spawn one acid projectile aimed at the player** (a lobbed arc). No touch damage.
  - Reuse HURT/DEAD exactly as melee.
- **Acid projectile = arc, so perches aren't safe.** A pooled `AcidProjectile` given an initial velocity that lobs toward the player's current position under a projectile gravity (ballistic). Because it arcs, a player on a ledge above the spitter can be hit — the perch exploit is answered by a real threat, not just RETREAT. On player overlap → `player.takeDamage(acidDamage, projectile.x)`, deactivate, emit a green splat. Expire on ground/terrain contact or lifetime.
- **LOS stays simple.** Fire if the player is within `firingRange` and on the side the spitter faces. No terrain raycasting for now (note as a future refinement) — don't over-engineer.
- **Placeholder art (golden rule 4):** generate a green blob for the spitter body, a small green projectile, and a green splat particle in `BootScene`. Real art swaps in later with zero logic change.

## Changes by file
1. **`src/config.js`**
   - Add `aiProfile` to each existing enemy entry = `'melee'` (explicit; behavior unchanged).
   - Add a **spitter** enemy entry: `{ aiProfile:'ranged', maxHealth, moveSpeed, detectionRadius, preferredRange:{min,max}, firingRange, attackCooldown, scrapDrop (unused until P3.3, ok to include), body, sheet:placeholder }`.
   - Add `acid: { speed, gravityY, damage, lifespan, poolSize, bodyRadius }` (projectile tuning; gravityY gives the arc).
   - Add placeholder colors to `palette` (spitter green, acid green) and a spitter `ENEMY`-style placeholder dimension.
   - Add **one spitter** to `LEVEL.enemies` positioned on the ground **below/near a floating ledge** so the perch test is real, e.g. `{ x: <under a ledge>, y: 524, type: 'Spitter' }`.
2. **`src/scenes/BootScene.js`**
   - Generate placeholder textures: `spitter` blob (green), `acid` projectile (small green disc), `acid-splat` particle. Keep existing generation intact.
3. **`src/entities/AcidProjectile.js`** (NEW — mirror `Bullet.js`)
   - `Physics.Arcade.Sprite` in a pooled Group. `fire(x, y, vx, vy)` sets position/velocity and enables a projectile **gravity** (`body.setGravityY(CONFIG.acid.gravityY)`) for the arc. `preUpdate` deactivates on lifespan elapsed or out-of-world. `deactivate()` stops/hides and clears gravity.
4. **`src/entities/Enemy.js`**
   - In `spawn()`, set `this.aiProfile` from config (default `'melee'`).
   - In `preUpdate`'s FSM, branch by profile: keep the existing melee CHASE/ATTACK; add a `'ranged'` path — CHASE kites to `preferredRange`, ATTACK fires via `this.scene.spawnAcid(this, player)` on cooldown (no touch damage). RETREAT/HURT/DEAD shared.
   - `updateAnimation`: ranged uses the same `walk` (kiting) and `attack` (spit) anim keys.
5. **`src/scenes/GameScene.js`**
   - Create `this.acid = this.physics.add.group({ classType: AcidProjectile, maxSize: CONFIG.acid.poolSize, runChildUpdate: true })`.
   - Add `spawnAcid(enemy, player)`: compute a lobbed velocity toward the player and `acid.get(...).fire(...)`; play the enemy's attack anim.
   - `physics.add.overlap(this.player, this.acid, this.onAcidHitPlayer, null, this)` → `player.takeDamage`, projectile `deactivate()`, splat particle.
   - Optional: `physics.add.collider(this.acid, this.terrain, splatOnGround)` so acid splats on the ground instead of rolling.
