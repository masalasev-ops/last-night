# P3.6 — Level system + second biome (Level 2)

> First milestone of Phase 3b (content). Turn the single hardcoded `CONFIG.LEVEL` into a **`LEVELS` data table keyed by `levelIndex`**, and have `GameScene` load the level the P3.5 cursor points at — finally *consuming* the `levelIndex`/`phase`/`nextLevelId` plumbing P3.5 built. Then author **Level 2** in a second biome (apocalyptic ruins, from the post-apocalypse backgrounds pack) with **zone-triggered spawns** so enemies appear as you advance instead of all at `create()`, paced as **tension → lull → tougher fight** rather than a uniform sprinkle. The **Flyer debuts here**. End state: `Level 1 (forest) → shop → Level 2 (ruins)` flows from data, and adding a Level 3 later is a new `LEVELS` entry, not engine work.

## Context
P3.5 built the cursor; P3.6 is where it earns its keep. Today:
- **`CONFIG.LEVEL`** is one object (`worldWidth`, `groundY`, `spawn`, `platforms[]`, `endMarkerX`, `pickups[]`, `enemies[]`). `GameScene.create()` reads it directly, and `update()` reads `CONFIG.LEVEL.endMarkerX`. **`runState.levelIndex` is never consulted** — so `advanceLevel()` (shop Continue) increments the cursor but the next level is always Level 1 (the documented "stub").
- **Biome art is global**: `buildParallax()` loops a fixed set of `bg-layer-${i}` keys (forest, 4 layers); `buildTerrain()` uses the single `tileset` key. Both are hardcoded, one biome.
- **Spawns are static**: `create()` loops `LEVEL.enemies` and spawns every enemy up front (they exist from the start, just far away). No pacing, no zone triggers.
- **Available second-biome art**: `public/assets/free-post-apocalypse-pixel-art-backgrounds-for-game-projects/` — several backgrounds, each with layered PNGs (`1/2/3.png`) that work as parallax layers. No separate ruins *ground* tileset ships, so Level 2 reuses the terrain tileset (optionally tinted) — distinct ground tiles are a later art drop.

So P3.6's core is a **level loader keyed by the cursor** plus a **zone-spawn system**, then Level 2 as data on top. The enemy roster, weapons, economy, and save all already work across a level transition (proven in P3.5) — this milestone gives the transition a genuinely *different second level* to land on.

## Scope guard
**In P3.6:**
- **`CONFIG.LEVEL` → `CONFIG.LEVELS`** — a map keyed by `levelIndex` (`1`, `2`). Each entry is full level data plus **biome fields** (`bgLayers`, `tileset`/tint) and **`nextLevelId`**.
- **`GameScene` loads `CONFIG.LEVELS[runState.levelIndex]`** (fallback to level `1`) — the single wiring that makes the P3.5 cursor real. All `LEVEL.*` reads (terrain, parallax, spawn, endMarker, pickups) come from the resolved level.
- **Per-biome assets** — `BootScene` preloads both biomes' parallax layers + tilesets under biome-scoped keys; `buildParallax()`/`buildTerrain()` read the resolved level's biome keys, and `buildParallax()` handles a **variable layer count** (forest 4, ruins 3).
- **Zone-triggered spawn system** — each level's enemies live in **`zones: [{ triggerX, enemies: [...] }]`**; `GameScene` fires a zone's spawns once when the player crosses its `triggerX`. Pooled enemies (`this.enemies.get`) as today.
- **Level 2 (ruins)** authored as data: new biome background, **its own new hand-authored layout** — a *different* `platforms` arrangement, `worldWidth`, `endMarkerX`, and encounter flow from Level 1 (NOT L1's geometry with a new background). L2 shares only the *systems* with L1 (the loader, zone-spawn mechanism, roster), not the level design; it should feel like a distinct place. Its own `pickups` and **paced `zones`** (tension → lull → tougher), with the **Flyer** in the mix.
- **`nextLevelId` routing** — `advanceLevel()` sets `levelIndex` to the current level's `nextLevelId`. Level 2's `nextLevelId` is `null` → Continue after clearing L2 hits a **"Chapter 1 — to be continued" placeholder** (the boss slots in here at P3.7).
- **L1 stays regression-safe** — Level 1's existing encounter is re-expressed as zones that reproduce its current feel (same types/positions), so it plays as it does today.

**NOT in P3.6:**
- **No boss** (P3.7) — L2 ends into a placeholder, not a boss. `nextLevelId: null` is the seam the boss plugs into.
- **No Flyer real art** — it debuts in L2 on its **placeholder blob** (its airborne-sprite problem is still parked). Gameplay only.
- **No new enemy types, weapons, or upgrades** — L2 uses the existing roster (now including the Flyer).
- **No distinct ruins ground tileset** — L2 reuses the terrain tileset (optionally tinted for mood); real ruins ground tiles are a later art drop via the same key.
- **No real menu/options** beyond the P3.5 Title (P3.8), **no audio** (P3.9), no cutscenes/dialogue (the survivor/clue is P3.7/P3.10).
- No mid-level checkpoints (P3.5's model stands: die → restart the current level from its start).
- No level editor / procedural generation — levels are hand-authored data.

## Design decisions
- **`LEVELS` keyed by `levelIndex`; `GameScene` resolves once.** `const LEVEL = CONFIG.LEVELS[runState.levelIndex] ?? CONFIG.LEVELS[1]` at the top of `create()`, and everything downstream reads that local `LEVEL` (as it already does — the diff is *where `LEVEL` comes from*, not the hundred uses). `update()`'s `endMarkerX` read switches to the resolved level too. This is the whole "consume the cursor" change; keep it a one-source swap, not a scatter.
- **Biome = a small bundle of asset keys on the level, not a hardcode.** Each level names its `bgLayers` (ordered far→near key list) and `tileset` key (+ optional `tilesetTint`). `BootScene` preloads forest keys (`bg-forest-0..3`, `tileset-forest`) and ruins keys (`bg-ruins-0..2`, reusing `tileset-forest` for ground). `buildParallax()` iterates `LEVEL.bgLayers` (variable length) instead of a fixed `i<4`. This keeps a third biome a data + asset-load addition, no `buildParallax` edit.
- **Zone-triggered spawns, fired once, pooled.** `LEVEL.zones` replaces `LEVEL.enemies`. `GameScene` tracks a `firedZones` set; each `update()`, any zone whose `triggerX <= player.x` and not yet fired spawns its enemies (via the existing pool) and is marked fired. A zone with `triggerX: 0` = "spawn at start" (how L1 keeps its current feel if desired). Enemies still spawn at ground/positions given per entry. No despawn-behind logic (out of scope) — zones only *add*.
- **Pacing is authored, not coded.** The engine just fires zones by position; the *rhythm* lives in how L2's zones are laid out: an opening skirmish, a quiet stretch, a mid build, a breather, then a climax cluster before the end marker. The spec gives L2 a concrete paced layout (below); it's tuning, refined in playtest.
- **`nextLevelId` drives progression; `null` = end-of-content.** `advanceLevel()` becomes `this.levelIndex = CONFIG.LEVELS[this.levelIndex]?.nextLevelId ?? this.levelIndex`. When the shop Continues from a level whose `nextLevelId` is `null` (L2 today), route to a **placeholder end scene/screen** ("Chapter 1 — to be continued") rather than looping. P3.7 replaces that with the boss.
- **Flyer debuts in L2, on its blob.** L2's zones include Flyer spawns (over a gap / above the player) — the airborne threat the perch-heavy geometry wants. Still the placeholder blob; real art stays parked.
- **L1 regression is the safety net.** Level 1 re-expressed as zones must reproduce today's encounter — same enemy types, counts, and positions (the Tank still guards the medic chest, the Spitter still sits in the mid-crossing). Verified against the current feel (test **B**).
- **Second biome reads distinct with what's on hand.** A different parallax background (apocalyptic ruins) + a tint on the reused ground tileset is enough to feel like a new place; don't block the milestone on bespoke ground art.

## Changes by file
1. **`src/config.js`**
   - Replace `LEVEL: { … }` with **`LEVELS: { 1: { … }, 2: { … } }`**. Level `1` = today's forest data, plus `biome`/`bgLayers`/`tileset` fields (the current forest keys), `nextLevelId: 2`, and its `enemies[]` re-expressed as **`zones`** that reproduce the current encounter.
   - Author **Level `2`** (ruins) as a **genuinely new map — a fresh hand-authored layout, not a copy of L1's `platforms`**: `biome: 'ruins'`, ruins `bgLayers`, reused `tileset` (+ `tilesetTint`), its **own distinct** `worldWidth`/`groundY`/`spawn`/`platforms` (different arrangement, length, and traversal challenge from L1)/`endMarkerX`/`pickups`, `nextLevelId: null`, and **paced `zones`** — e.g. Z1 opening (2 zombies), lull, Z2 build (Runner pair + a Spitter on a ledge), lull, Z3 climax (a Tank + a Flyer over the gap + a zombie) before the end marker.
   - Add a `BIOMES`/asset manifest (or extend `ASSETS`) listing each biome's bg-layer file paths + tileset so `BootScene` can preload them by key.
2. **`src/scenes/BootScene.js`**
   - Preload **both biomes'** parallax layers (`bg-forest-0..3`, `bg-ruins-0..2`) and tileset(s) from the manifest, under biome-scoped keys. (Forest keys keep their current names to avoid churn, or alias.)
3. **`src/scenes/GameScene.js`**
   - `create()`: resolve `const LEVEL = CONFIG.LEVELS[runState.levelIndex] ?? CONFIG.LEVELS[1]` and use it everywhere `CONFIG.LEVEL` was used. Initialize `this.firedZones = new Set()`; do **not** spawn all enemies up front.
   - `buildParallax()`: iterate `LEVEL.bgLayers` (variable count) using the level's keys.
   - `buildTerrain()`: use `LEVEL.tileset` (+ apply `LEVEL.tilesetTint` if set).
   - `update()`: **zone spawner** — for each `LEVEL.zones` not in `firedZones` with `triggerX <= this.player.x`, spawn its enemies (pooled) and add to `firedZones`. `endMarkerX` reads from the resolved `LEVEL`.
   - Win → shop path unchanged (`enterShop()` then Shop); the level identity is already captured by `levelIndex`.
4. **`src/runState.js`**
   - `advanceLevel()`: `this.levelIndex = CONFIG.LEVELS[this.levelIndex]?.nextLevelId ?? this.levelIndex;` (was `levelIndex++`); keep `phase='level'` + `checkpoint()`. When `nextLevelId` is `null`, `advanceLevel()` leaves the cursor and the caller routes to the end placeholder (see ShopScene).
5. **`src/scenes/ShopScene.js`**
   - Continue: if the current level's `nextLevelId` is `null` → route to the **end placeholder** (a minimal "Chapter 1 — to be continued" screen or the Title) instead of `start('Game')`; otherwise `advanceLevel()` + `start('Game')` as today.
6. **(Optional) `src/scenes/EndScene.js` (NEW, minimal)** — a placeholder "to be continued" text screen for clearing the final level, with a button back to Title. Kept trivial; P3.7 supersedes it with the boss/victory flow. (If simpler, reuse the Title with a one-line banner instead of a new scene.)
